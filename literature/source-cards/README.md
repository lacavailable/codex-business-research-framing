# Source cards

Each source has one Markdown card with YAML frontmatter conforming to
`../schemas/source-card.schema.json`. The body must be an original paraphrase,
identify the exact material actually reviewed, distinguish metadata from
content access, and state counterconditions or limits. Cards do not reproduce
article text.

A card becomes `verified` only after separate fresh-context extraction and
audit passes both record `pass` and all corresponding manifest fields agree.

Use all nineteen explicit level-two headings, in this order:

1. Bibliographic information
2. Source tier
3. Authority rationale
4. Inclusion rationale
5. Problem addressed
6. Central contribution
7. Supported framing principles
8. What it does not establish
9. DFC-12 relevance
10. Managerial-framing implications
11. Scholarly-positioning implications
12. Relevant output modes
13. Candidate Skill rules
14. Anti-patterns
15. Counterconditions
16. Exact evidence locations
17. Access and license status
18. Extraction confidence
19. Independent audit result

Keep every section within the material's recorded access scope. An explicit
heading does not authorize an inference unsupported by the reviewed content.
