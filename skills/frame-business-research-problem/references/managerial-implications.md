# Managerial Implications

Write each implication as a conditional decision statement with these fields:

| Field | Requirement |
|---|---|
| Actor | Name who can act. |
| Decision | Name the controlled choice. |
| Condition | State when the implication applies. |
| Mechanism | Explain how the choice changes the outcome. |
| Model-supported consequence | State only what the analysis establishes. |
| Evidence status | Label empirical fact, model assumption, simplifying assumption, inference, or unsupported claim. |
| Implementation requirement | Name data, authority, workflow, timing, or capability needed to act. |
| Excluded claim | State the tempting conclusion that is not established. |
| Boundary | State the principal scope condition or countercondition. |

Do not translate runtime reduction directly into realized profit, a private objective into social welfare, or a comparative static into a recommendation without the operational pathway. If no actor can implement the modeled choice before the relevant deadline, report the implication as unresolved.
