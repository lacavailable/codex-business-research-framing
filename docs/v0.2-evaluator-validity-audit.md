# v0.2 Evaluator Validity Audit

Audit date: 2026-07-20  
Baseline commit: `ef3a3c0bc65f3082b050df9067e8689db3d97bd3`  
Baseline release: `v0.2.0-beta`

## Reproducibility

The baseline was reproduced before any v0.3 changes. The 26-test suite passed, the installed official Skill validator passed, the published v0.2 package verified, and aggregation of the committed 126 outputs and 252 primary scores completed without missing pairs. The recomputed and committed summaries have the same SHA-256 digest:

`a5a88682ae2db7354523455360f47d2f336eefd76ebd427bac51224db700d540`

All v0.2 cases, prompts, outputs, scores, mappings, reports, and rubric files remain unchanged for provenance.

## Validity defects

1. **Generator exposure to judge-only fields.** `prepare()` serializes the complete v2 case into each generator prompt. That object includes fidelity requirements, prohibited inferences, evidence expectations, hard-failure traps, and other answer-key material. High fidelity scores therefore do not measure unaided defect recognition.
2. **Category applicability is documented but not implemented.** The v2 score schema contains one integer per category and no applicability field. `normalize_score()` and `reconcile_scores()` enforce every category minimum, even when the rubric says a layer is conditional.
3. **Premature rounding and category-wise median reconciliation.** Reconciliation rounds the median of each category separately. The resulting category vector need not equal any judge's coherent evaluation and discards decimal precision before aggregation.
4. **The adjudicator is not authoritative.** When adjudication is triggered, v2 takes a three-score median. An adjudicator can identify the correct interpretation yet be outvoted numerically by two incompatible primary scores.
5. **Severe ceiling effects.** Ninety-nine of 126 reconciled outputs scored at least 95. The compressed distribution cannot distinguish meaningful improvements reliably.
6. **Low judge total-score correlation.** Primary total-score correlation was 0.151. Scholarly positioning had the largest mean absolute category disagreement (2.183 points), and 35 of 126 outputs required adjudication.
7. **Unequal length and structure.** No-Skill outputs averaged 134.1 words, v0.1 outputs 381.3, and v0.2 outputs 309.6. Only v0.1 consistently used Markdown headings. Content, length, and structure effects are confounded.
8. **Same-model-family generation and judging.** Generators, primary judges, and adjudicators came from the same model family. Blind IDs concealed condition labels but not stylistic signatures, leaving response-process and contamination risk.
9. **OR-specific regressions.** v0.2's OR mean was 92.643, below its OM and IS means. Against no Skill, prominent negative pairs included `or-11` (-10); against v0.1, `or-11` and `or-13` were both -8.
10. **Hard-failure reliability is overstated by prevalence.** Agreement was 0.992 because judges agreed that 125 outputs had no failure and disagreed on the only asserted failure. Positive and negative agreement were not reported separately.
11. **Possible encoding and judge-perception artifacts.** The committed JSON is valid UTF-8 and contains legitimate Unicode punctuation. Windows console rendering displayed mojibake during inspection, demonstrating that judge or operator interfaces can distort otherwise valid text. v3 must validate code points and rendered packet transport, not only JSON parsing.

## Supported and unsupported interpretations

The v2 run supports artifact completeness and deterministic aggregation. It does not support claims that the rubric is calibrated, that total scores are reliable, that conditional layers are scored correctly, or that v0.2 is superior to the baselines. The v3 evaluator must establish separate content, response-process, internal-structure, external-criterion, contrast-sensitivity, and consequential evidence before its scores are used for Skill optimization.
