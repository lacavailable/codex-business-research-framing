# Source Selection and Exclusion Log

## Included core

The fixed v0.2 corpus contains 20 complementary sources: four practitioner guidance or execution observations, six management and organization sources, three Information Systems sources, six Operations Management or Operations Research sources, and one bounded scholarly-rhetoric source. Canonical citations, access scope, selection rationale, and limitations are in [`literature/manifest.yaml`](../literature/manifest.yaml).

Selection favors authority, direct relevance, complementarity, and a material contribution to the rule registry. Inclusion does not endorse every claim in a source.

## Metadata-only and access-limited treatment

No source is treated as fully reviewed unless the manifest and audited card identify lawful full-text access. DOI metadata, publisher abstracts, summaries, and official landing pages support only the claims visible there. All v0.2 entries use metadata-and-original-notes-only redistribution.

|---|---|---|
| Additional HBR exemplars | Excluded for v0.2 | The three selected execution observations already cover the intended bounded practitioner role; more would add publisher concentration without changing a rule. |
| MIT Sloan Management Review exemplars | Excluded for v0.2 | No candidate was shown to add a distinct rule beyond the focused core. |

| Additional INFORMS editorials | Deferred | Add only when they contribute a distinct implementation or decision-aiding rule not represented by the fixed corpus. |

## Change control

Any later inclusion must update the manifest, source card, independent audit, rule support, generated matrix, notices, and validation. A redistributable file additionally requires an explicit permissible license, checksum, attribution, license record, and notice agreement.
