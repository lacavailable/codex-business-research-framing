# Skill 2.2 adaptive root-cause audit and frozen design plan

## Preserved starting point

This branch starts from PR #7 commit
`323ea17b8f98059b71801b6a1bf9241188be4fee`. PRs #5, #6, #7, and #8 remain
open, draft, unmerged historical records. Their candidate files, calls,
results, attestations, and sealed artifacts are not rewritten. Both official
holdouts remain unopened.

## Read-only findings

1. Skill 2.1 routed profile before a precise deliverable contract, so compact
   and standard answers could include the wrong layers even when their prose
   was sound.
2. Its nominal minimum word counts conflicted with useful short outputs and
   encouraged compliance judgments unrelated to content sufficiency.
3. Repetition was prohibited in principle but lacked semantic merge steps and
   a single visible home for each limitation.
4. Table suppression was treated too much like concision, although the IS
   timing/information task benefits from a compact mapping.
5. Status rules did not sharply separate an explicit contradiction from absent
   information, and PR #8 later showed that polished compression can upgrade
   unsupplied semantics into “stated” semantics.
6. Claim scope lacked an explicit singular/plural and benchmark-generalization
   audit.
7. Domain routing alone was insufficient for formulation, algorithmic,
   analytical, causal, governance, and mechanism-design contributions.
8. The old unsupported-number and repetition metrics measured lexical form
   rather than unsupported factual claims or substantive restatement.
9. Full-audit rendering exposed internal completeness and repeated boundaries
   even when a smaller audit would preserve every safeguard.

## Decision-complete intervention

- Route deliverable before profile.
- Use adaptive `micro`, `compact`, `standard`, and explicit-only `full-audit`
  contracts with soft maxima and no padding minimum.
- Apply a seven-question minimum-sufficient filter, semantic single-home
  deduplication, and adaptive table/prose selection.
- Make status, claim granularity, contribution type, candidate-setting class,
  managerial-value links, and clarification value explicit.
- Preserve DFC-12, six gates, BusinessBrief 1.0/2.0 validation, the validator,
  and OpenAI agent metadata.
- Add a product-only deterministic canary summarizer that distinguishes
  unsupported empirical or generalized claims from structural numbers. It does
  not alter or calibrate an evaluator.
- Freeze the Skill tree, eight reused task requests, eight frozen v0.2 outputs,
  prompts, manifest, metrics, and all fourteen acceptance rules before any
  authoritative call.

## Task tuple and budget

- **M0:** PR #7 head `323ea17b8f98059b71801b6a1bf9241188be4fee`.
- **Objective:** improve targeted usability, organization, economy, claim
  precision, and evidence boundaries without weakening fidelity.
- **E_dev:** deterministic Skill, package, repository, privacy, copyright, and
  link checks; zero model calls.
- **E_test:** one eight-task public development canary against the already
  frozen v0.2 outputs. This is not independent validation or either official
  holdout.
- **Authoritative budget:** exactly eight candidate generations and eight
  single-judge blinded comparisons; no baseline regeneration, retries,
  replacements, adjudication, or additional model roles.

The canary is a noncompensatory merge gate for this draft development PR only.
It cannot authorize validation, a holdout, a performance claim, a tag, or a
release.
