# v0.1 to v0.2 Change Analysis

## Preserved

- The Skill name remains `frame-business-research-problem`.
- DFC-12 is unchanged.
- Actor, timing, information, behavior, constraints, and objective remain noncompensatory gates.
- Failed fidelity remains ineligible regardless of writing quality.
- Examples remain synthetic unless supported by user-supplied evidence.

## Added

- Four input-readiness stages and an explicit `not_assessed` state.
- Literature-grounded managerial-framing and scholarly-positioning layers.
- A 20-source canonical manifest, two-pass cards, rule registry, and generated traceability matrix.
- BusinessBrief 2.0 while retaining BusinessBrief 1.0 validation.
- A canonical five-layer, 100-point rubric and generated human reference.
- Twelve harder benchmark cases and a three-condition v0.2 comparison design.
- Expanded user documentation, bilingual invocations, licensing controls, and release checks.

## Policy corrections

The hand-maintained v0.1 Markdown threshold conflicted with the machine-readable rubric. v0.2 archives the v1 JSON, makes one v2 JSON canonical, generates its Markdown, and uses a 70-point threshold with a 69-point hard-failure cap. Later-layer scores cannot offset a fidelity hard failure or an applicable layer minimum.

## Compatibility

Existing 1.0 briefs remain valid through the version-aware validator. New work should use 2.0 so unknown information, readiness, eligibility, and layer audits are represented explicitly.
