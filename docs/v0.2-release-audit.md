## Release decision

**Local release candidate approved for pull request.** Tagging and publication remain conditional on the merged `main` commit, required GitHub CI, and post-publication archive verification. This document does not claim that the release has been published.

The evaluation itself does not show aggregate superiority: v0.2's mean capped score was 95.310, compared with 96.000 for no Skill and 95.738 for v0.1. The complete report preserves that negative result.

## Literature and copyright

All 20 sources have separately produced extraction and audit records with conservative access scopes and evidence locations. `literature/manifest.yaml` is canonical. No third-party PDF, HTML snapshot, screenshot, or full text is included. `literature/open-access/` and `literature/licenses/` contain policy READMEs only, and `THIRD_PARTY_NOTICES.md` records that no third-party full text is bundled.

Repository audits reject tracked private-corpus paths, document/archive/credential extensions, and oversized textual artifacts except for narrow policy-controlled cases. The ignored `research-private/` path is skipped by `--all` but would be rejected if tracked.

## Evaluation caveat

The forward test is complete but has a strong ceiling effect, low total-score judge correlation, and same-model-family judging risk. v0.2 avoided reconciled hard failures but regressed on aggregate score. See [`v0.2-evaluation-report.md`](v0.2-evaluation-report.md). Release eligibility means the protocol is complete and honestly reported; it does not mean the beta won the benchmark.

## External publication checklist

1. Push the branch without force-pushing and open the pull request.
2. Wait for all required CI checks and inspect the hosted diff and commit history.
3. Merge with a merge commit, preserving the ten logical commits; do not delete the branch.
4. Tag the merged `main` commit as `v0.2.0-beta` only after rerunning the release gates.
5. Verify the prerelease contains the archive, checksum, changelog, installation guide, evaluation Markdown/JSON/status, literature manifest, notices, and source-selection log.
6. Download the published archive and checksum, verify SHA-256, extract to a fresh directory, run official validation, and confirm every release URL and asset.
