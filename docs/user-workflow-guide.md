# User Workflow Guide

## Choose the maturity ceiling

Start with the [research input template](../skills/frame-business-research-problem/assets/research-framing-input-template.md). Use `unknown` instead of guessing. Exploratory framing can diagnose missing structure; model-grounded framing is the primary use; evidence-grounded and manuscript-ready work require progressively stronger inputs.

## Choose a mode

- Use `create` for one fidelity-tested setting.
- Use `diagnose` to separate fidelity, managerial, scholarly, evidence, and prose weaknesses.
- Use `rewrite` when source text exists and changes must be categorized.
- Use `compare-scenarios` for multiple candidate settings; ineligible candidates are never ranked.
- Use `map-model-to-business` to expose semantic mappings and gaps.
- Use `audit-assumptions` to separate tractability from business interpretation.
- Use `draft-introduction` only when positioning and evidence support it.
- Use `draft-managerial-implications` only when results have an actor-specific operational pathway.

## Interpret the result

`eligible` means all six fidelity gates pass on supplied information. `ineligible` means at least one gate fails. `not_assessed` means the evidence is insufficient to decide. None of these states validates the mathematical model or establishes empirical truth.

Review the mapping, evidence labels, boundaries, and primary remaining weakness before using drafted prose. Resolve unknown essential fields before treating an exploratory result as manuscript-ready.
