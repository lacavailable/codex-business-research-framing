# Security Policy

## Supported versions

Until the first stable release, only the newest published prerelease receives security fixes.

## Reporting a vulnerability

Do not open a public issue for a vulnerability, exposed credential, private-corpus leak, archive traversal problem, or prompt-injection path involving bundled resources. Use GitHub's private vulnerability reporting feature for this repository. If that feature is unavailable, contact a maintainer privately through their GitHub profile and request a secure reporting channel without including sensitive details in the first message.

Include the affected version, impact, minimal reproduction, and suggested mitigation when possible. Expect an acknowledgment within seven days. No response-time or remediation guarantee is implied.

The Skill provides writing and research-framing assistance, not security, legal, financial, or operational guarantees. Treat generated claims as unverified until independently checked.
